# M8.1 使用手册 — MCP Streamable HTTP 与工具刷新

> 面向终端用户 / 新人上手。M8.1 在 M8 stdio MCP 基础上新增 HTTP MCP server 接入与 `tools/list_changed` 自动刷新。

---

## 1. 前置条件

- Bun >= 1.3；
- 已配置 `NOVA_API_KEY` 或 `~/.nova-code/config.json.apiKey`；
- 若 HTTP MCP server 需要认证，把 token 放进环境变量，不建议直接写明文：

```bash
export MCP_TOKEN='your-token'
```

---

## 2. 添加 Streamable HTTP MCP server

```bash
nova-code mcp add-http remote \
  --header 'Authorization=Bearer ${MCP_TOKEN}' \
  --timeout-ms 10000 \
  https://mcp.example.com/mcp
```

可信 server 可显式跳过审批：

```bash
nova-code mcp add-http trusted-remote \
  --auto-approve \
  --header 'Authorization=Bearer ${MCP_TOKEN}' \
  https://mcp.example.com/mcp
```

查看配置：

```bash
nova-code mcp list
nova-code config get
```

`config get` 会把 headers 脱敏：

```json
{
  "mcpServers": {
    "remote": {
      "type": "http",
      "url": "https://mcp.example.com/mcp",
      "headers": { "Authorization": "****" }
    }
  }
}
```

---

## 3. 查看 MCP 工具

```bash
nova-code mcp tools
```

输出示例：

```text
MCP__remote__search MCP server 'remote' tool 'search'.
```

工具名仍统一使用：

```text
MCP__<server>__<tool>
```

---

## 4. 在 ask / chat 中使用

```bash
nova-code ask "用 remote MCP 工具查询一下项目相关信息"
nova-code chat
```

安全默认：

- MCP 工具默认进入权限审批；
- `--auto-approve` 只应给你完全信任的 server 使用；
- HTTP headers / stdio env 不会出现在 `config get` 明文输出中。

---

## 5. `tools/list_changed` 行为

若 MCP server 声明并发送：

```text
notifications/tools/list_changed
```

nova-code 会自动重新调用该 server 的 `tools/list` 并刷新 bridge 后的工具列表。

当前生效边界：

- chat：下一轮用户输入前会读取最新工具列表；
- ask：单次运行启动时固定工具列表，中途刷新不会改变已经发给模型的 tool schema；
- 刷新失败只显示/记录 warning，不影响其他 MCP server 与内置工具。

---

## 6. 端到端本地验证脚本

HTTP MCP server 真实接入依赖具体服务。仓库内自动化用 unit fixture 覆盖协议链路；你可以运行：

```bash
bun test src/services/mcp/McpStreamableHttpClient.test.ts
bun test src/services/mcp/mcpToolRegistry.test.ts
bun test src/commands/McpCommand/McpCommand.test.ts
```

完整提交前校验：

```bash
bun run typecheck
bun test
bun run check
```

---

## 7. 故障排查

| 现象 | 可能原因 | 处理 |
|---|---|---|
| `HTTP 401` | header/token 未生效 | 确认 `MCP_TOKEN` 已 export；`config get` 只显示脱敏值 |
| `unsupported content-type` | server 没按 MCP Streamable HTTP 返回 JSON 或 SSE | 检查 server endpoint 是否是 MCP endpoint |
| `timed out` | server 无响应或网络慢 | 增大 `--timeout-ms` |
| `No MCP tools available` | server `tools/list` 为空或启动失败 | 先跑 `nova-code mcp tools` 看 warning |
| chat 里刚新增工具不可见 | 通知刚到、刷新尚未完成 | 下一轮输入前通常会可见；必要时重启 chat |
