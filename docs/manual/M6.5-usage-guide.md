# M6.5 — Phase 1 收官使用手册

> 适用版本：v0.7.x（M6.5 上线之后）
>
> 面向：终端用户 / 新人上手 / 维护者

---

## 1. 升级后必跑

```bash
bun install
bun run typecheck
bun test
bun run check
```

---

## 2. sessionId 行为变化

新建 `chat` 会话时，欢迎横幅中的 session 现在是 UUID v4：

```text
nova-code chat（session: 3f4e2b70-8f4a-4d47-9e4f-2c3b7f7a8e10, model: claude-sonnet-4-5-20250929）
```

已有历史会话仍可加载：

```bash
nova-code chat --resume 2026-05-04T10-00-00-deadbeef
nova-code chat --resume my-old-alias
```

M6.5 只改变新 ID 的生成，不迁移也不重写 `~/.nova-code/sessions/*.jsonl`。

---

## 3. 运行性能 baseline

```bash
bun run perf:baseline
bun run perf:baseline -- --runs=10
```

输出包含 JSON（stdout）和人类可读表格（stderr）。当前 Phase 1 baseline 见 [docs/performance/M6.5-baseline.md](../performance/M6.5-baseline.md)。

---

## 4. 离线验证 M3 权限主路径

```bash
bun test src/m6-5-e2e-phase1.test.ts
```

该测试不访问真实 Anthropic API，覆盖：

1. ask 默认 `acceptEdits`：`FileEdit` 放行，`Bash` 在 headless 下 deny；
2. `--dangerously-skip-permissions`：普通 `Bash` 直接执行；危险命令黑名单仍由权限单测覆盖。
